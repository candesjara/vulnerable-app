# Matriz de Vulnerabilidades

Guia de evaluacion para docentes. El proyecto es intencionalmente vulnerable y debe ejecutarse solo en laboratorio local.

| ID | Categoria | Archivo afectado | Descripcion de la vulnerabilidad | Herramienta que podria detectarla | Tipo de analisis | Severidad estimada | Referencia OWASP Top 10 | Posible mitigacion |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| VULN-001 | Autenticacion insegura | `backend/src/controllers/authController.js` | Comparacion directa de password en texto plano contra el valor almacenado en base de datos. | SonarQube, Semgrep, revision manual | SAST/manual | Alta | A07: Identification and Authentication Failures | Usar hash fuerte con salt, por ejemplo bcrypt o Argon2. |
| VULN-002 | Autenticacion insegura | `backend/src/db/init.js` | Usuario administrador creado por defecto con credenciales conocidas. | Semgrep, revision manual | SAST/manual | Alta | A07: Identification and Authentication Failures | Crear flujo de bootstrap seguro y forzar cambio de password inicial. |
| VULN-003 | Autenticacion insegura | `backend/.env`, `backend/.env.example`, `README.md` | Credenciales y secretos debiles incluidos en archivos del proyecto. | GitLeaks, TruffleHog, Semgrep, revision manual | SAST/manual | Alta | A02: Cryptographic Failures | No versionar secretos y usar gestion segura de secretos. |
| VULN-004 | Enumeracion de usuarios | `backend/src/controllers/authController.js` | Mensajes de error diferentes revelan si el usuario existe o si solo fallo el password. | OWASP ZAP, Burp Suite, revision manual | DAST/manual | Media | A07: Identification and Authentication Failures | Usar respuestas genericas para fallos de autenticacion. |
| VULN-005 | Sesion insegura | `backend/src/controllers/authController.js` | Token predecible construido con ID, username y rol. | Semgrep, revision manual | SAST/manual | Alta | A07: Identification and Authentication Failures | Usar tokens firmados robustos, expiracion corta y claims minimos. |
| VULN-006 | Sesion insegura | `backend/src/middleware/auth.js` | El middleware acepta tokens sin firma ni validacion criptografica. | Semgrep, SonarQube, revision manual | SAST/manual | Alta | A01: Broken Access Control | Validar firma, expiracion, issuer y audience del token. |
| VULN-007 | Sesion insegura | `frontend/src/pages/Login.jsx`, `frontend/src/services/api.js` | Token y usuario completo se guardan en `localStorage`. | Semgrep, ESLint Security, revision manual | SAST/manual | Media | A07: Identification and Authentication Failures | Usar cookies `HttpOnly`, `Secure`, `SameSite` y reducir datos persistidos. |
| VULN-008 | Inyeccion SQL | `backend/src/controllers/authController.js` | Consulta de login concatena `username` directamente en SQL. | Semgrep, SonarQube, ESLint Security | SAST | Critica | A03: Injection | Usar consultas parametrizadas y validacion de entrada. |
| VULN-009 | Inyeccion SQL | `backend/src/controllers/clientController.js` | Creacion de clientes concatena campos del body en sentencia `INSERT`. | Semgrep, SonarQube | SAST | Critica | A03: Injection | Usar prepared statements y esquema de validacion. |
| VULN-010 | Inyeccion SQL | `backend/src/controllers/clientController.js` | Busqueda de clientes concatena `q` en filtros `LIKE`. | OWASP ZAP, Semgrep, SonarQube | SAST/DAST | Critica | A03: Injection | Parametrizar busquedas y escapar comodines si aplica. |
| VULN-011 | Inyeccion SQL | `backend/src/controllers/clientController.js` | Consulta directa por ID usa `req.params.id` sin parametrizar. | Semgrep, SonarQube, OWASP ZAP | SAST/DAST | Alta | A03: Injection | Validar tipo numerico y usar parametros. |
| VULN-012 | Inyeccion SQL | `backend/src/controllers/invoiceController.js` | Registro de facturas concatena todos los campos recibidos. | Semgrep, SonarQube | SAST | Critica | A03: Injection | Usar prepared statements y validacion por tipo. |
| VULN-013 | Inyeccion SQL | `backend/src/controllers/invoiceController.js` | Busqueda por numero y monto concatena parametros `number` y `min`. | OWASP ZAP, Semgrep | SAST/DAST | Alta | A03: Injection | Parametrizar filtros y validar rangos numericos. |
| VULN-014 | XSS almacenado | `frontend/src/pages/Clients.jsx` | Observaciones de clientes se renderizan con `dangerouslySetInnerHTML`. | Semgrep, SonarQube, revision manual | SAST/manual | Alta | A03: Injection | Sanitizar HTML o renderizar texto plano. |
| VULN-015 | XSS almacenado | `backend/src/db/init.js`, `backend/src/controllers/clientController.js` | Se permite guardar HTML y payloads en `notes` sin sanitizacion. | OWASP ZAP, Semgrep, revision manual | SAST/DAST/manual | Alta | A03: Injection | Validar entrada y sanitizar salida con libreria confiable. |
| VULN-016 | Control de acceso roto | `backend/src/routes/adminRoutes.js`, `backend/src/controllers/adminController.js` | Endpoints administrativos no exigen rol real de administrador. | OWASP ZAP, Semgrep, revision manual | DAST/manual | Critica | A01: Broken Access Control | Aplicar autorizacion server-side por rol en cada endpoint sensible. |
| VULN-017 | Control de acceso roto | `frontend/src/components/Nav.jsx` | El boton Admin se oculta en frontend segun rol, pero el backend no protege el recurso. | Revision manual, OWASP ZAP | DAST/manual | Alta | A01: Broken Access Control | No depender del frontend para autorizacion. |
| VULN-018 | IDOR | `backend/src/controllers/clientController.js` | Cualquier usuario puede consultar clientes cambiando el ID en la URL. | OWASP ZAP, Burp Suite, revision manual | DAST/manual | Alta | A01: Broken Access Control | Verificar propiedad o permisos sobre el recurso solicitado. |
| VULN-019 | Exposicion de informacion sensible | `backend/src/controllers/adminController.js` | `/api/admin/debug` expone variables de entorno, headers, memoria y directorio de trabajo. | OWASP ZAP, Semgrep, revision manual | SAST/DAST/manual | Alta | A05: Security Misconfiguration | Eliminar endpoints debug o restringirlos con autenticacion fuerte. |
| VULN-020 | Exposicion de informacion sensible | `backend/src/controllers/adminController.js` | `/api/admin/config` devuelve secreto JWT, usuario y password admin. | OWASP ZAP, Semgrep | SAST/DAST | Critica | A02: Cryptographic Failures | No exponer secretos y separar configuracion interna de respuestas API. |
| VULN-021 | Logging inseguro | `backend/src/controllers/authController.js`, `backend/src/middleware/auth.js`, `frontend/src/services/api.js` | Logs imprimen passwords, tokens, datos personales y sesion completa. | SonarQube, Semgrep, revision manual | SAST/manual | Media | A09: Security Logging and Monitoring Failures | Redactar secretos y definir politica de logging segura. |
| VULN-022 | Errores verbosos | `backend/src/app.js`, varios controladores | Respuestas de error incluyen `stack`, SQL o mensajes tecnicos. | OWASP ZAP, SonarQube, revision manual | SAST/DAST/manual | Media | A05: Security Misconfiguration | Usar manejo centralizado de errores con mensajes genericos. |
| VULN-023 | CORS inseguro | `backend/src/app.js` | CORS permite origen `*`, headers amplios y metodos innecesarios. | OWASP ZAP, Semgrep | SAST/DAST | Media | A05: Security Misconfiguration | Restringir origenes, metodos y headers requeridos. |
| VULN-024 | Upload inseguro | `backend/src/controllers/uploadController.js` | `multer` acepta archivos sin validar extension, MIME type ni tamano. | Semgrep, OWASP ZAP, revision manual | SAST/DAST/manual | Alta | A05: Security Misconfiguration | Validar tipo, extension, tamano y analizar contenido. |
| VULN-025 | Upload inseguro | `backend/src/controllers/uploadController.js`, `backend/src/app.js` | Archivos se guardan con nombre original y se sirven publicamente desde `/uploads`. | Semgrep, OWASP ZAP | SAST/DAST | Alta | A05: Security Misconfiguration | Renombrar archivos, aislar almacenamiento y controlar descarga. |

| VULN-027 | Componentes vulnerables | `frontend/package.json` | Dependencias antiguas o innecesarias usadas en frontend. | npm audit, Snyk | SAST | Media | A06: Vulnerable and Outdated Components | Actualizar librerias y remover dependencias no usadas. |
